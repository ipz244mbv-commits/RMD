# Лабораторна робота №1: Основи React Native та Expo

Виконав: Макарчук Богдан Дисципліна: Розробка мобільних додатків

## Опис проекту

Цей проект є базовим мобільним застосунком, створеним за допомогою фреймворку React Native та інструментарію Expo. Додаток містить реалізовану навігацію за допомогою `@react-navigation/bottom-tabs` та складається з трьох основних екранів:

1. Головна: Відображає стрічку новин за допомогою компонента `FlatList`.
2. Фотогалерея: Демонструє відображення зображень у вигляді сітки з двох колонок, використовуючи динамічне завантаження фотографій за URL.
3. Профіль: Містить форму реєстрації користувача з використанням `TextInput`, `ScrollView` та базової валідації стану (useState).

На всіх екранах присутній спільний заголовок та підвал із даними розробника.

## Інструкція із запуску

1. Встановіть залежності проекту:

npm install

Запустіть сервер розробки Expo:

npx expo start

Відскануйте згенерований QR-код за допомогою камери вашого iOS пристрою (або додатком Expo Go для Android).

Основні способи запуску додатку в Expo
Під час розробки Expo пропонує кілька варіантів з'єднання між комп'ютером та мобільним пристроєм:

LAN: - Призначення: Стандартний спосіб запуску. Пристрої зв'язуються напряму через вашу локальну Wi-Fi мережу.

Особливості: Найшвидший спосіб передачі даних.

Відмінності: Комп'ютер та телефон обов'язково мають бути підключені до одного роутера без ізоляції клієнтів.

Tunnel

npx expo start --tunnel

Призначення: Використовується, коли пристрої знаходяться в різних мережах або локальна мережа блокує прямі з'єднання.

Особливості: Expo створює тимчасову публічну URL-адресу. Додаток завантажується через інтернет.

Відмінності: Працює стабільно за будь-яких умов, але завантаження та оновлення коду відбувається значно повільніше, ніж через LAN.

Емулятор / Симулятор (Localhost):

Призначення: Тестування без фізичного пристрою.

Особливості: Запуск на віртуальному Android (через Android Studio) або iOS (через XCode).

Відмінності: Зручно для перевірки верстки на різних розмірах екранів, але вимагає значних ресурсів комп'ютера (ОЗП, процесор) для роботи віртуальної машини.
